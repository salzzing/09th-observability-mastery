## 3. 중요하다고 생각한 개념

이번 주 내용 중 중요하다고 생각한 개념 2~3개를 골라 정리해주세요.

### 개념 1. 관측 가능성(Observability)

#### 개념 설명

> Observability는 시스템이 외부로 출력하는 Metrics, Logs, Traces 같은 데이터를 이용해서 시스템 내부 상태를 이해하는 개념이다.
단순히 CPU 사용률이나 에러 발생 여부를 보는 수준이 아니라, 왜 문제가 발생했는지, 어떤 요청에서 문제가 생겼는지, 앞으로 어떤 장애가 발생할 가능성이 있는지까지 파악하는 데 목적이 있다.

#### 중요하다고 생각한 이유

> 실제 서비스는 여러 서버, 애플리케이션, 네트워크, 데이터베이스, 컨테이너, 클라우드 리소스가 함께 동작하기 때문에 단일 지표만으로는 장애 원인을 찾기 어렵다.
예를 들어 CPU 사용률이 높다는 사실만으로는 어떤 영향을 미쳤는지, 사용자에게 어떤 문제가 발생했는지까지는 알 수 없다.
Observability는 여러 데이터를 연결해서 시스템을 더 입체적으로 볼 수 있게 해준다는 점에서 중요하다고 생각했다.

#### 운영 관점에서의 의미

> 운영자는 장애가 발생했을 때 빠르게 원인을 좁혀야 한다. 관측 가능성이 잘 갖춰져 있으면 Metrics로 이상 징후를 감지하고, Logs로 상세 이벤트를 확인하며, Traces로 특정 요청의 흐름을 추적할 수 있다.
결과적으로 장애 대응 시간이 줄어들고, 반복 장애를 예방하거나 자동화된 운영 체계를 구축하는 데도 도움이 된다.
